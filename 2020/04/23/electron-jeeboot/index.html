##jeecg boot与electron结合构建桌面应用##

构建项目脚手架

采用electron-vue脚手架来构建项目，按照官方文档（https://simulatedgreg.gitbooks.io/electron-vue/content/cn/）
如果没有按照vue-cli的话需要先安装vue-cli，生产脚手架代码如下：

vue init simulatedgreg/electron-vue my-project

然后按照向导一步一步完成项目的初始化。完成后进入项目目录执行如下代码：

npm install
npm run dev

如果一切正常的话，可以看到electron 的欢迎界面。

jeecg boot 代码集成

electron在src文件夹下生成了两个主要的目录分别为main和renderder，其中main为主进程操作的目录，renderer为渲染进程相关的代码。集成jeecg boot主要是将jeect boot相关代码移植到renderder目录下。将renderder目录下原有的文件全部删除，然后将jeecg boot前端项目（ant-design-vue-jeecg）src文件夹下的内容全部拷贝到renderder目录下，然后运行npm run dev ,然后你回看到控制台会有一大堆的错误，主要有如下几类问题：

  • 找不到module

此类问题是因为缺少了相关的npm包，根据jeecg boot的package.json文件，将缺少的module添加到项目中。

  • scss less相关问题

如果不能识别less、scss，检查package.json中是否有相关loader的依赖。同时由于在less中启用了javascript，因此需要开启相关配置。在项目的.electron-vue目录下找到webpack.renderer.config.js ,添加或修改关于less的配置。

{test: /\.less$/,
use: [
  'vue-style-loader',
  'css-loader',
  {
loader: 'less-loader',
options: { 
  javascriptEnabled: true 
}
  }
]
  }
  • jsx语法的支持

这里我安装了babel-plugin-syntax-jsx和babel-plugin-transform-vue-jsx插件，然后项目目录下.babelrc文件增加了对jsx的支持。这里一直比较疑惑jeecg boot 是怎么支持的

  • 资源的加载
    jeect boot里面配置了一些alias引用，详见vue.config.js,这需要在webpack.renderer.config.js 进行相关配置。

-dev环境下http请求处理

需要在.electron-vue下的dev-runner.js中配置代理

const server = new WebpackDevServer(
  compiler,
  {
contentBase: path.join(__dirname, '../'),
quiet: true,
proxy:{
  '/jeecg-boot': {
target: 'http://localhost:8088', //请求本地 需要jeecg-boot后台项目
ws: false,
changeOrigin: true
  },
},
before (app, ctx) {
  app.use(hotMiddleware)
  ctx.middleware.waitUntilValid(() => {
resolve()
  })
}
  }
)

经过以上几类问题的处理后再次运行，会报一个tag组件的错误在（sMenu组件内），发现代码里tag是一个变量，一直没有找到合适的办法解决，最终直接修改了tag为其真实的值router-link。

项目打包

运行npm run build后回报javascript heap memory 类似内存不足的错误，百度了一下需要在构建的命令行node后面增加–max_old_space_size=8192
参数（值应该还可以少一点，暂未测试）。
http请求访问了本地路径问题，修改utils目录下的request.js,如果不是开发环境路基为完整的路径。

const baseURL = process.env.NODE_ENV === 'development'
  ? '/jeecg-boot'
  : 'http://192.168.199.205:8088/jeecg-boot'
// 创建 axios 实例
const service = axios.create({
  baseURL: baseURL, // api base_url
  timeout: 15000 // 请求超时时间
})
